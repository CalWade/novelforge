# 都市言情·铁律补充（iron-laws-extra）

> 本文件是 `rules/24-iron-laws.md` 的题材补充，由 Evaluator 在审 urban-romance-contemporary 题材章节时额外加载。
> Evaluator 引用时使用 `iron_law_extra_N` 标识符。
> 仅适用于都市言情（职场 + 情感克制 + 成年人选择）题材。

---

## iron_law_extra_1 · 情感推进必须有心理成本

成年人的情感不是"相遇→心动→在一起"的直线，而是"权衡→犹豫→选择"的过程。任何感情质变（告白/牵手/接吻/明确交往/分手/复合）必须在 500 字内看到主角的"心理成本"。

**判据**：本章发生了感情节点（牵手 / 接吻 / 明确交往 / 同居 / 分手 / 复合 / 表白）。

**hit 条件**：
- 节点前后 500 字内**未出现**纠结、犹豫、代价权衡、自我说服的内心独白或具体动作（捏杯子、按电源键、在笔记本上画两列）
- 或节点由外部催化（朋友撮合 / 酒后 / 意外事件 / 车祸）触发，主角全程被动接受
- 或节点发生时主角的语言是「我爱你」「你是我的命」这类直白宣告，而非通过"不回消息/点开对话框又关掉/把手放在中央扶手上"这种行为承载

## iron_law_extra_2 · 职业描写禁止"背景板化"

主角写的是产品经理、研发、投行、VP——这些职业不能只在名片上出现。每涉及职场场景必须落地到具体工作细节，禁止"假行话"。

**判据**：本章出现工作场景（开会 / 评审 / 汇报 / 对外见客户 / 加班 / 离职谈判 / 晋升沟通等）。

**hit 条件**：
- 出现「她在开会」「他在写代码」这种笼统白描，未给出**具体会议类型**（需求评审/OKR复盘/季度述职/跨部门拉齐）
- 产品经理沈若微说的话里**没有**"排期/优先级/MVP/埋点/用户画像/竞品/需求文档"这类词汇
- 创业者林昭宇说话像"CEO霸总体"而非工程师出身的务实语气（张口闭口"战略格局""赛道""风口"而不谈产品和代码）
- 投行顾安安提到工作时出现明显不专业的表述（把"卖方"说成"投行方"、把"DD"说成"调研"、把"TS"说成"意向书"等外行话）
- 晋升/跳槽/竞业等决策被写成"一拍脑袋"而非"能力×时机×有人说话"三要素同时满足的结果

## iron_law_extra_3 · 物价与消费必须对齐 2024-2026 一线城市

era.md 锚定的时代是 2024-2026 中国一线城市（深圳为主）。所有金额、房租、工资、消费水平必须与此相符，禁止穿越感。

**判据**：本章出现金额数字（工资 / 房租 / 外卖价 / 打车费 / 股权 / 融资额等）。

**hit 条件**：
- 一线城市产品经理月薪写成 8000 元（对应 2010 年代初）或 30 万（对应顶级总监级，与沈若微"高级产品经理"不符）
- 深圳科技园一居室房租写成 2000 元（对应二线城市）或 3 万（对应豪宅）
- 外卖价格写成 10 元以下（对应 2014-2016）或 200 元以上（不符合日常）
- 打车费出现"起步价 10 块"（2015 年水平）或莫名其妙的"打车花了 300 元"（市内不符）
- 林昭宇的天使轮融资金额写成"5000 万美元"（与 AI 客服早期创业不匹配，应在 500 万-2000 万人民币区间）
- 咖啡、星巴克、瑞幸等品牌的价格明显跑偏（瑞幸标价 35 元 / 星巴克标价 15 元均为错误）

## iron_law_extra_4 · 必须使用本土 App 与通讯方式

时代是 2024 年中国大陆，通讯、支付、出行、社交、点餐必须用本土产品。写成"WhatsApp / Instagram / Uber / Facebook Messenger / Google Maps"视为穿越到海外或 2010 年代。

**判据**：本章出现通讯、支付、打车、点餐、导航、社交媒体场景。

**hit 条件**：
- 通讯用 WhatsApp / iMessage / Line / Telegram 而非**微信**
- 社交媒体用 Instagram / Twitter / Facebook / TikTok 而非**小红书 / 微博 / 抖音 / 朋友圈**
- 打车用 Uber / Lyft 而非**滴滴 / 高德打车 / 美团打车**
- 外卖用 Deliveroo / DoorDash 而非**美团外卖 / 饿了么**
- 支付用 Apple Pay / PayPal / 刷卡为默认而非**微信支付 / 支付宝**
- 导航用 Google Maps 而非**高德 / 百度地图**
- 沈若微打开"企业微信"的场景写成 Slack、写"飞书"的场景写成 Microsoft Teams
- 工作协同出现 Jira / Confluence 而非**飞书文档 / 钉钉 / 语雀 / Tapd**（外企或特定情境例外，但需有合理解释）

## iron_law_extra_5 · 情感落点必须具体化

严禁"一股暖流/心跳漏了一拍/世界仿佛静止/眼眶一热/鼻子一酸"等陈词滥调。情感推进必须落在**具体动作、物件、环境变化**上。

**判据**：本章描写情感波动（心动 / 难过 / 纠结 / 感动 / 愤怒 / 失落等）。

**hit 条件**：
- 出现以下任一高疲劳词/短语：「心跳加速」「心跳漏了一拍」「脸颊发烫」「脸红了」「一股暖流」「世界仿佛静止」「时间静止了」「眼眶湿润」「鼻子一酸」「心里一颤」「沦陷了」「无法自拔」
- 情感波动**未**通过具体动作/物件承载（捏杯沿、按三次电源键、把筷子放下、在笔记本上画线、把手放在中央扶手上、关掉又打开对话框）
- 情感描写是"直接宣告"而非"侧面呈现"（例：「她意识到自己喜欢上他了」= hit；「她把他昨天发的消息截屏存进了一个只有自己能看的相册」= 不 hit）

## iron_law_extra_6 · 禁止玛丽苏/杰克苏/霸总套路

主角不得拥有没铺垫的巨额财富、顶级人脉、全能才艺；感情进展不得走"合约婚姻/假戏真做/壁咚/女人你成功引起了我的注意"等套路。任何"爽点"必须有清晰代价。

**判据**：本章出现感情或身份的戏剧化展开。

**hit 条件**：
- 出现霸总腔台词：「女人 / 小东西 / 乖 / 闭嘴，听我说 / 你逃不掉的 / 我说过你是我的」
- 出现玛丽苏场景：无铺垫的壁咚、无铺垫的公主抱、无铺垫的"他为了她放弃一切"、车祸救美、英雄救美于小巷
- 主角突然被揭示"其实是某隐藏富二代/真千金/海归精英后代"且此设定在前文无任何伏笔
- 感情线出现：合约婚姻 / 假扮情侣 / 契约交往 / 带球跑 / 失忆 / 绝症 / 替身文学 / 车祸失忆三连
- "所有异性角色都喜欢主角"（同章内出现 2+ 个男配对沈若微有明显暧昧意图且无独立动机）
- 主角做重大决定（辞职 / 迁居 / 放弃晋升 / 花光积蓄）的理由是"为了他 / 为了爱"而**没有**与事业/家庭/自我价值的权衡过程

## iron_law_extra_7 · 配角必须有独立意志

闺蜜顾安安、同事赵恺、VP 季凛、沈母——每个人必须有自己的利益诉求、恐惧、判断，不得纯粹为"推沈若微的剧情"而存在。

**判据**：本章出现配角参与的场景（对话 / 行动 / 决策）。

**hit 条件**：
- 配角的对话完全服务于主角（只问主角的事、只给主角意见、没有自己的烦恼与进展）
- 闺蜜顾安安出场只为"分析沈若微的感情"，没有自己的工作 / 婚姻 / 孕期焦虑推进
- VP 季凛的所有决策都是"在考验沈若微 / 在为沈若微铺路"，而非基于他自己的 KPI 与人事棋盘
- 沈母出现只为"催婚 / 施压"，没有她作为退休教师的日常生活质感
- 配角的情绪与判断永远**同步主角**（主角难过配角就来安慰、主角开心配角就来庆祝），缺少"我今天也有自己的事"的独立性

## iron_law_extra_8 · 感情节奏硬约束

前 3 章不表白、不牵手、不拥抱、不接吻；第 5 章之前不明确在一起；全书不写"我爱你"三个字（除非有极其充分的铺垫）。此条为 setting 级强制节奏，Evaluator 不得放行。

**判据**：检查当前章节号与感情推进事件。

**hit 条件**：
- ch1-ch3 内出现牵手 / 拥抱 / 接吻 / 告白 / 明确交往
- ch1-ch4 内出现"沈若微与林昭宇（或任何男配）已经在一起"的明确状态
- 全书任何章节主角脱口而出"我爱你"且前文没有至少 3 章的铺垫（共同经历的危机、多次选择的累积、关键信任事件）
- 感情推进速度超出「认识三周约会、认识两月同居、认识半年求婚」这类快节奏边界（本题材禁止韩剧/偶像剧节奏）
